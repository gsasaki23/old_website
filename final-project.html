User Testing

  User Stories

  • User stories are a type of User Testing that I adapted to get a general understanding of what my users wanted. This was accomplished by comparing my interface with my feature lists, and then looking at my website from the perspective of different types of users.
  • Please see here for a full view of the document.

  Cognitive Walkthrough

  • A cognitive walkthrough is a type of User Testing that I adapted to determine if a user can complete a set of tasks walking through a working prototype of my website. It helps to identify issues regarding navigation, content and aesthetics.
  • While conducting these, it was rather difficult to not run these like User Interviews where users can throw back any comments back at me, but rather to have the users only do the tasks that I asked them to do.
  • Please see here for a full view of the document.

  Task-Based User Tests

  • Two User Tests were ran using UsabilityHub.com.
  • The first was a question test, which asked how the user would contact the website owner. This checked the functionality and organization of the content of the Contact page. Results show that both the email and phone number were easy to locate.
  • The second was a simple click test, which asked where a contact email address would be found from the home page. This looked for two things: 1) To see if the naming of the buttons on the navigation toolbar were appropriate, and 2) To see if it made sense for the contact email to belong on the Contact page. Results show that both were designed correctly.
  • Please see here to view the details of the first test, and here to view the details of the second test.

Summary of Findings

      The primary issue that arose from user research and testing was the lack of content and design, which was leading to the website being very boring to look at. Each page was perceived as too plain and bland, especially due to the lack of images throughout the website.

      Another major issue that was raised was the location of the resume. All interviewees agreed that the resume did not belong in the Contact page. One suggestion was to move the image link to another page. Another suggestion was to make it a page of its own with other projects that I have worked on, calling it "Experience" or something along those lines.

      The last major issue was the Facts page. In general, interviewees were not too impressed with the idea of a button that generates random facts. It was also found that the button did not work on certain browsers such as Firefox. It was suggested to be replaced by the titles of the facts, which could then drop down to show more text and images when clicked.

      Throughout conducting user research and tests, as the creator, it was very difficult for me to be biased in my opinions on the website. I found myself constantly trying to defend features and design components while talking to my interviewees. It was very interesting because I did not expect myself to act this way when I was planning out the interview questions.

      Before conducting the user research and testing, I already expected to get negative feedback on the lack of content in general, but I was not expecting the same for the resume and its significance on the website. I also now believe that many of the layout and design choices were from a severe lack of front-end knowledge, rather than poor planning.

Incorporation of Findings

    Notable changes that have been made to the website since the initial submission include:
  • Adding this "Research" page, which shows and explains the User Research, User Testing, and Summary and Incorporation of Findings since the initial submission.
  • Making the website no longer automatically refresh every minute.
  • Moving the resume to the now renamed "Interests" page, and showing it on the page instead of just linking to the image.

      In the future, I plan to make many changes to the overall design of the website. Many of these changes would come from simply learning more about the coding aspect of websites rather than making better design choices.

    Major future changes include:
  • Creating a new "Experiences" page, which would include most of my work experience and examples of coding projects that I have worked on. The link to the Resume image would also be moved here.
  • Replacing the "Facts" button with another way to display all of the facts, their text, and images.

  • Correcting breakpoints.
